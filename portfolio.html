MCPFolio

A fully local AI-powered stock analysis and portfolio management tool. Runs entirely on your machine — no cloud, no telemetry, encrypted storage.

MCPFolio

Full-stack local AI finance app. A FastAPI/FastMCP backend exposes 8 MCP tools; a LangGraph ReAct agent orchestrates Ollama-powered analysis; a Vite/Chart.js frontend renders a dashboard, portfolio tracker, and streaming chat advisor.

Architecture:

  • Backend: FastMCP (stdio) + FastAPI — 8 tools covering market snapshots, sentiment, news, earnings, macro context, and sector peers
  • AI Layer: LangGraph ReAct reasoning loops driving Ollama local inference (llama3.2 default, hot-swappable at runtime)
  • Frontend: Vite + vanilla JS with Chart.js — dashboard, encrypted portfolio tracker, and streaming chat advisor with tool-call visibility

Key Features:

  • 30-day trend charts and composite risk scoring (volatility, sentiment, P/E)
  • AES-256 encrypted portfolio storage (PBKDF2HMAC) — password never persists to disk
  • Parallel prefetching (3–6 concurrent tool calls) for fast analysis
  • Zero telemetry — only external call is yfinance for market data

EngComp Autograder

Built to support Engineering Computations I & II at GWU. Provides an end-to-end local grading pipeline without requiring a JupyterHub server.

Workflow:

  • Instructor authors assignments in Jupyter Notebooks using nbgrader formatting
  • Assignments generated and distributed via Blackboard LMS
  • Student submissions collected as a .zip with an accompanying .csv gradebook
  • Autograded locally — results displayed and saved to a scratch directory

Includes automatic nbgrader directory structure setup, formgrader compatibility, and handles the full create-distribute-collect-grade cycle.

Tocify

A fork of the original tocify project with transformative architectural changes. The original was a flat collection of scripts that committed markdown digests back to the repo. I completely reimagined it with a structured codebase, email delivery, and a provider-agnostic AI layer.

Core Architectural Shifts:

  • Repo Restructure: Reorganized the original flat-file layout into a clean src/ package (core engine, email layer, LLM wrapper, feed checker) and a dedicated settings/ folder (feeds, interests, LLM config, prompt template) — separating runnable code from user-customizable config.
  • Delivery Model Redesign: Original system → commit markdown digests to repo. My fork → generate structured JSON data and deliver beautiful HTML emails to inbox instead. This eliminates repository bloat from weekly digest commits while providing a superior user experience.
  • Multi-Provider LLM Abstraction: Built a generic llm_wrapper.py layer supporting Anthropic, OpenAI, and Google Gemini interchangeably — provider and model configurable via secrets or settings/llm.json.

Supporting Enhancements:

  • Email Design: Built responsive HTML email templates with card-based layout, color-coded relevance badges, source attribution pills, and mobile optimization
  • Parser Robustness: Enhanced RSS parsing to handle diverse date formats across different journal feeds

Skills Demonstrated:

  • Codebase Organization: Imposed clear structure on an unorganized proof-of-concept, separating concerns across src/ and settings/
  • System Architecture: Redesigned delivery pipeline to solve a fundamental problem (repo bloat)
  • API Abstraction: Designed a provider-agnostic wrapper supporting multiple LLM vendors behind a unified interface
  • Email Engineering: Built production-ready responsive HTML emails

Boltzmann 0D Solver

A self-contained educational toolkit for exploring the Boltzmann transport equation under an applied electric field, developed as a final project for ECE 6030 at GWU. Designed to make kinetic theory accessible through interactive, annotated simulation.

Two Solver Approaches (side-by-side):

  • Deterministic (RK4): Fourth-order Runge-Kutta integration of the relaxation-time Boltzmann equation — precise and ideal for building physical intuition.
  • Stochastic (Monte Carlo): Particle-based sampling that converges to the same solution statistically — illustrates the equivalence between kinetic and probabilistic descriptions.

Educational Design:

  • Theory grounded in theory.ipynb — derivations, approximations, and physical motivation alongside the code
  • Side-by-side RK4 vs. Monte Carlo results to build intuition for both approaches
  • Structured for reuse by students learning plasma or gas-phase kinetics
